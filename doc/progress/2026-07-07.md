## 2026-07-07 — PLINK2 PCA fix (all-zero eigenvec from PLINK1)

- **Diagnosis:** Published `process/{mod}/pca/*.pca.eigenvec` and `.eigenval` were all zeros (e.g. `D.pca.eigenvec` lines `0 ABD_0001 0 0 …`); plot showed a single point at (0,0) because input data was degenerate, not a parser/plot bug.
- **Cause:** PLINK1 `--pca` on full cohort (~7700–8300 samples) exits 0 but returns zero eigen decomposition; manual `plink2 --pfile … --pca approx 10` on same data yields non-zero PCs (e.g. D eigenvals 576.593, 133.724, …).
- **Fix:** `plink1_pca` process now runs `plink2 --pfile {prefix} --mind 0.9999 --pca approx ${params.pc_num}` on merged `*_test.plink2`; tmp entries `plink1_pca_replot.nf` and `ibs_pca_replot.nf` use `listMergedSubgenomeTestPfileTuples` for PCA (IBS still plink1 bfile).
